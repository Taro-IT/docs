P23 - Proceso de release de línea base

Objetivo(s)#

  • Permitir generar un release de las líneas base definidas del departamento.

Entradas#

  • Repositorio de la línea base a desplegar.
  • La rama con todos los ítems de trabajo previamente verificados y aprobados.

Si no se tiene claro lo que es una línea base, revisar la política de elementos de configuración y líneas base.

Procedimiento#

FaseActividadesÁrea(s) involucradas
1.Identificación.
  • Verificar que haya ocurrido al menos una de las siguientes:
    • Se acerca una fecha de entrega con los socios o del SCAMPI.
    • La rama de develop ha adquirido suficientes funciones como para presentarlas en una demo.
    • Se ha completado un epic o módulo completo de historias de usuario.
  • VER - 1.2
  • OPF - 3.1
2. Preparación.
  • Crear la rama de release siguiendo la guía de deployment Frappé o la guía de release de Gitflow.
  • En caso de ser un despliegue de la wiki seguir la guía de release de la wiki.
  • Corregir los comentarios generados para el release en caso de ser necesario.
  • Crear un respaldo dentro de una carpeta con el nombre del release en el respaldo de releases.
  • Agregar al folder una subcarpeta con toda la documentación para el socio.
  • CM - 3.2
  • OPF - 3.1
  • OPF - 3.2
3. Despliegue.
  • Desplegar la página en el ambiente de producción siguiendo el manual de despliegue de Espresso o en el caso de Frappé y la wiki departamental, una vez concluidas las pruebas se hará el despliegue de forma automática.
  • CM - 1.3
  • OPF - 3.1

Salida#

  • Release de línea base publicada.
